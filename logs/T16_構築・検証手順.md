# T16：案件パトロール自動化 ── 構築・検証手順（社長用）

- 作成日：2026-07-14
- 担当：⚙️ ハック
- 使うファイル：`logs/T16_案件パトロール自動化.gs`（これ1つだけ）
- 所要時間：構築 約10分 ＋ 接続テスト 1分 ＋（うまくいけば）毎朝の確認1〜2分

---

## 0. 最初に読んでください（いちばん大事な注記）

T7の「毎朝ブックマーク8個をクリックする手動パトロール」を自動化してほしい、というご依頼で作りました。
ただし正直にお伝えする必要があります。

**この作業環境（ハックが今コードを書いている場所）からは crowdworks.jp に一切接続できません。**
接続しようとすると403（アクセス拒否）が返ってきます。これはT7・T9のときと同じ制限です。

今回、次の2つを追加で確認しました：

1. 別のネットワーク経路（Anthropicのフェッチ基盤）からも試しましたが、やはり403でした。
2. さらに、`crowdworks.jp/robots.txt`（本来はどのサイトでも誰でも読める、認証不要の公開ファイルです）に
   アクセスしても403でした。これは「特定のページだけ守っている」のではなく、**入口（WAF）の段階で
   機械的なアクセスをまとめてブロックしている**可能性を示しています。
3. また調べたところ、クラウドワークスの新着案件RSS配信は**2018年12月に終了済み**（公式アナウンス確認済み）で、
   代替になる公開APIも見当たりませんでした。

**Google Apps Script の `UrlFetchApp` は Google のサーバーから実行されるため、ハックの作業環境とは別経路になり、
上記の制限を受けない可能性はあります。** ただし「動くはず」であって「確実に動く」ではありません。
実際に動くかどうかは、**社長にスプレッドシート上で接続テストを実行して確認していただく必要があります**
（これがこの手順書の一番大事なステップです＝パート2）。

もしGAS側でもブロックされていた場合は、パート4の「次善策（判断だけの自動化）」に切り替えてください。

---

## パート1：構築（約10分・PCで）

### STEP 1：新しいスプレッドシートを作る
1. ブラウザで https://sheets.new を開く
2. そのまま次へ

### STEP 2：コードを貼り付ける
1. メニューの「拡張機能」→「Apps Script」をクリック
2. 最初から入っている数行（`function myFunction() {...}`）を全部消す
3. `logs/T16_案件パトロール自動化.gs` の中身を全部コピーして貼り付け
4. 💾（保存）アイコンをクリック。プロジェクト名は「案件パトロール」など適当でOK

### STEP 3：setup を実行する
1. 関数選択プルダウンで **`setup`** を選ぶ
2. 「▶ 実行」をクリック

### STEP 4：承認画面を進める（初回だけ）
1. 「承認が必要です」→「権限を確認」
2. 自分のGoogleアカウントを選ぶ
3. 「このアプリは Google で確認されていません」→「詳細」→「案件パトロール（安全ではないページ）に移動」
4. 「許可」をクリック
5. 「セットアップが完了しました！🔍」のダイアログが出れば成功

### STEP 5：できあがりを確認する
- [ ] シートが6枚：**はじめに／案件ログ／設定／実行ログ／手動貼付判定／生HTML保存**
- [ ] 「設定」シートに検索URL8本（★1〜3はチェック済み・☆4〜8は未チェック）が入っている
- [ ] 「設定」シートに即×ワード・◎ワードのリストが入っている
- [ ] メニューバーに「🔍案件パトロール」が出る（数秒待つ・再読み込みで出ることも）

---

## パート2：接続テスト（1分・**これが今回いちばん大事な確認**）

1. メニュー「🔍案件パトロール」→「🧪 接続テスト（1件だけ・まずこれ）」を実行
2. 出てくるダイアログを見る：

   **✅「接続できました！」と出た場合**
   - HTTPステータス200＋案件が実際に何件か検出されていれば成功です。パート3へ進んでください。
   - ただし「0件でした」という注記がついた場合、**接続はできているのにページの中身が想定と違う**
     （＝クラウドワークスがページの見た目・構造を変えていて、こちらの読み取りルールが古い）状態です。
     この場合は「生HTML保存」シートに保存された内容をコピーして、ハックにそのまま共有してください。
     読み取りルール（正規表現）を実際のHTMLに合わせて直します。

   **❌「接続できませんでした」と出た場合**
   - クラウドワークス側でbotアクセスとしてブロックされています。「実行ログ」シートにHTTPステータスと
     エラー内容が残っているので、念のため確認してください。
   - この場合、**自動取得は諦めます**（無理に回避策を探ると規約違反のリスクがあるため）。
     パート4「次善策」に切り替えてください。

**社長が最初に確認すべきこと（繰り返しになりますが重要）**：
「動いた」というダイアログの言葉だけでなく、**「案件ログ」シートに実際にそれらしい案件タイトルと
実在しそうなURL（`crowdworks.jp/public/jobs/数字`）が入っているか**を必ず目視してください。
タイトルが文字化けしていたり、URLが変だったりする場合は「動いているように見えて中身が壊れている」
状態なので、その画面のスクリーンショットもハックに共有してください。

---

## パート3：本番運用（接続テストが成功した場合）

1. メニュー「🔍案件パトロール」→「🔎 今すぐ全件パトロール実行」で試しに1回、全★☆分を回してみる
   （☆4〜8は初期設定でOFFなので、まずは★1〜3だけが動きます）
2. 「案件ログ」シートを確認：
   - 判定＝「要チェック◎」（緑）の行を優先してチェック
   - 判定＝「除外」（グレー）の行は即×ワードに該当したもの。消してはいないので後から見返せます
3. 問題なさそうなら、メニュー「⏰ 毎朝7時の自動実行をON」でトリガーを登録
   （承認画面がもう一度出ることがあります。「詳細」→「移動」で進めてください）
4. 以降は毎朝、スプレッドシートを開いて「案件ログ」の新着（要チェック◎）を見るだけです。
   有望案件があれば、これまでどおりT9のチェックリスト形式でリサに精査を依頼してください。

**最初の3〜4日は毎朝、実際にクラウドワークスのサイトを開いて答え合わせをすることを強くおすすめします。**
自動取得が急に0件になったり、明らかにおかしなタイトルが増えたりしたら、クラウドワークス側の
ページ変更でブロック・構造変化が起きたサインです。その時点で「生HTML保存」シートの中身をハックに
共有してください。

---

## パート4：次善策（接続テストが失敗した場合）

**「取得」の自動化は諦めても、「判断（30秒足切り）」の自動化はできます。**

1. これまでどおりT7の手順で、社長がブラウザでブックマーク8個を目で見て巡回する
2. 気になった案件があれば、「手動貼付判定」シートの3行目以降に
   **タイトル（必須）／URL（できれば）／報酬（分かれば）** を貼り付ける（何件でもまとめてOK）
3. メニュー「📋 手動貼付シートを判定する」を実行
4. 「判定」列に「除外」「要チェック◎」「通常」が自動で入る（T7の即×ワード・◎ワードのルールで判定）

これで「巡回」は人がやりつつ、「即×ワードに引っかかっていないか」の確認作業だけ自動化できます。
T7の5分ルーティンのうち、判断に使っていた時間を短縮する狙いです。

## 知っておいてほしい仕様（設計判断のメモ）

| 項目 | 仕様 | 理由 |
|---|---|---|
| 解析方法 | HTML内の `/public/jobs/数字` リンクを起点に前後のテキストを正規表現で抜き出す | 本文はページ構造依存になりがちだが、案件詳細URLの形式（数字ID）は変わりにくいと考えられるため、これを軸にした |
| 解析の検証状況 | **未検証**（作業環境からアクセス不可のため、実際のHTMLを見て確認できていない） | 正直な制約。接続テスト・生HTML保存シートで社長側に検証してもらう設計にした |
| 重複防止 | 「案件ログ」のURL列を見て、既出URLはスキップ | シンプルな仕組みで十分な規模のため |
| 除外行 | 消さずにグレー表示で残す | あとで見返したいときのため（除外基準が厳しすぎたかの判断材料にもなる） |
| 実行時刻 | 毎朝7時（Googleの時間主導トリガーの仕様上、前後15分程度の誤差あり） | T7の「毎朝の習慣」の時間帯に合わせた |
| エラー処理 | 沈黙で失敗させず、必ず「実行ログ」に記録 | 依頼の必須要件 |
| 手動貼付判定 | 自動取得がダメだった場合の次善策として同梱 | RSS終了・API無しのため、取得の完全な代替手段が無い中での現実的な落としどころ |

## よくあるトラブル

| 症状 | 対処 |
|---|---|
| メニュー「🔍案件パトロール」が出ない | ページ再読み込み→数秒待つ |
| 接続テストで403・エラー | パート4の次善策へ切り替え |
| 接続OKなのに0件・タイトルが変 | 「生HTML保存」シートをハックに共有→読み取りルールを調整 |
| トリガーが動いていない気がする | Apps Scriptエディタ左メニューの「トリガー」アイコンから、`patrol` のトリガーが有効か確認 |
| シートを壊した | 壊れたシートを削除→メニュー「🛠 初期セットアップ」を再実行（既存シートには触らない安全設計） |
