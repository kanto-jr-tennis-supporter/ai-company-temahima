## 0. この設計書の考え方（要件定義メモ）

顧問業務の「手間」を数字で押さえた。v1で削るのはこの3つ：

| 業務 | 誰が | 頻度 | 現状の手間（控えめ見積） | キット導入後 |
|---|---|---|---|---|
| 出欠確認・記録・欠席者の把握 | 顧問 | 毎日（週5〜6） | 1回10分 ≒ **月200分** | 1回2〜3分（一覧チェック→自動転記） |
| 部費・会計の記録と年度末報告 | 顧問（会計担当） | 記帳は月2〜3回＋年度末 | 月30分＋**年度末報告に2〜3時間** | 記帳のみ。報告書は自動集計 |
| 大会エントリー（名簿抜き出し・参加費計算・引率名簿） | 顧問 | 年5〜8大会 | 1大会60〜90分 | 1大会15分（チェックを入れるだけ） |

合計で**月3〜5時間＋年度末の丸1晩ぶん**を削る。これが商品の約束であり、盛らない上限。

T8の戦略条件も設計に反映済み：
- 単品では売らない。**「出欠＋会計＋大会＋連絡文」の一式**で有料の土俵に乗せる
- 無料アプリ（BAND等）との差別化＝**保護者にアプリを入れさせない。顧問1人のスプシで完結**
- 「現役の部活顧問が自分の部で使っているもの」という当事者性を、商品そのものの作り（現場仕様のプルダウン等）で証明する

## 3. 各シートの機能要件（ハック向け・実装粒度）

> 実装原則（全シート共通）：
> - **IMPORTRANGE・外部API・アドオンは使わない**（権限承認で非IT教員が確実に詰まる）。1ファイル内で完結。
> - GASは**カスタムメニュー（onOpenで「🎾部活キット」メニューを追加）**からボタン実行。トリガーの自動設定は初回メニュー実行時に案内。
> - 数式で済むものは数式（ARRAYFORMULA/COUNTIF系）で組み、GASは「転記・行列生成・シート生成」だけに絞る（壊れにくさ優先）。
> - 生徒が触る想定はゼロ。**入力者は顧問1人**。編集範囲保護は「名簿」「設定」シートのみ警告付き保護。

### A. 出欠管理シート（基本版・核）

社長の既存シート（テニス部運用中）の構成を汎用化する。

**A-1.「名簿」シート（唯一の初期設定場所）**

| 列 | 項目 | 型・入力規則 |
|---|---|---|
| A | 学年 | プルダウン（1〜3。中高両対応のため「設定」シートで4〜6も選択可に） |
| B | 組 | 手入力 |
| C | 番号 | 手入力 |
| D | 氏名 | 手入力（保護者からの名簿貼り付けを想定し、姓名分割はしない） |
| E | 担任 | 手入力（任意） |
| F | 備考 | 任意（既往症・注意事項など。ヘッダーに「⚠️個人情報。記載は最小限に」注記） |
| G | 在籍 | プルダウン（在籍/引退/退部）── 引退・退部者を削除せず非表示化するため |

- 20行の見本データ入り（架空名。「この見本を消して貼り替えてください」と1行目に案内）
- 最大60名まで対応（ARRAYFORMULAの参照範囲は60行で固定し、それ以上は「設定」で拡張手順を案内）

**A-2.「今日の出欠」シート（毎日触る画面。スマホ閲覧・入力に耐えるシンプル構成）**

| 列 | 項目 | 仕様 |
|---|---|---|
| A〜D | 学年・組・氏名・担任 | 名簿から自動参照（ARRAYFORMULA。「在籍」のみ表示） |
| E | 区分 | **プルダウン：出席／欠席／遅刻／早退／公欠／見学**（候補は「設定」シートの一覧を参照。学校ごとに書き換え可能） |
| F | 理由 | 手入力（欠席・遅刻・早退・公欠のとき） |

- 条件付き書式：欠席=赤系、遅刻・早退=黄系、公欠=青系、見学=グレー（区分の値で行に薄く色）
- 未入力（空欄）は「出席扱いにしない」。集計上は「未記録」として別カウント（つけ忘れが見える）
- ヘッダーに日付セル（初期値 =TODAY() だがGAS転記時に値化）

**A-3.「出欠簿」シート（月間マトリクス・自動生成）**

- 行=部員（名簿連動）、列=活動日。**GAS「✅ 今日の出欠を記録する」**で以下を実行：
  1. 「今日の出欠」のE・F列を、「出欠簿」の当日日付列に記号で転記（出=○/欠=×/遅=遅/早=早/公=公/見=見/未記録=空欄）
  2. 当日列が無ければ右端に自動追加（日付ヘッダー・曜日付き）
  3. 転記後、「今日の出欠」のE・F列をクリアして翌日に備える（クリア前に確認ダイアログ）
- 月が替わったら新しい月ブロックを自動で区切る（列グループ or 月見出し行）。実装が重ければ「1シート=1年度・列を月で色分け」まで簡略化してよい
- 転記のやり直し：同日に再実行したら上書き確認を出す

**A-4. 集計・アラート（数式でつくる。「出欠サマリ」シート）**

- 部員別：出席数／欠席数／遅刻・早退数／公欠数／出席率（公欠は分母から除外——ここは学校現場の慣習。数式にコメントで明記）
- 期間切替：当月／年度累計の2表
- **連続欠席アラート**：直近の活動日3回連続「×」の部員を氏名リストで上部に表示＋出欠簿上で赤強調（生徒指導・保護者連絡のトリガーになる、顧問目線の機能）
- 「未記録が3日以上続いています」の自己アラート（顧問のつけ忘れ防止）

**A-5.「設定」シート**

- 区分プルダウンの候補一覧（編集可）、学年の範囲、部活名、年度開始月（4月固定でも可だがセルに切り出す）
- ここ以外はユーザーに数式を触らせない設計

### B. 会計簿シート（完全版）

**B-1.「会計簿」シート（記帳）**

| 列 | 項目 | 仕様 |
|---|---|---|
| A | 日付 | 手入力 |
| B | 種別 | プルダウン（収入／支出） |
| C | 費目 | プルダウン（収入：部費・補助金・その他／支出：消耗品・大会参加費・交通費・登録料・その他。「設定」で編集可） |
| D | 内容 | 手入力 |
| E | 金額 | 手入力（数値） |
| F | 残高 | 自動（累計数式） |
| G | 領収書 | プルダウン（あり／なし）── 監査対応の実務目線 |

**B-2.「部費徴収」シート**

- 行=部員（名簿連動）、列=徴収回（第1回〜。徴収額と締切をヘッダーに入力）
- チェックボックスで納入管理。**未納者は自動で赤色＋シート上部に「未納者一覧」を数式で自動表示**（そのまま声かけリストになる）
- 徴収合計は会計簿に自動では書き込まない（二重計上事故防止。「合計を確認して会計簿に1行記帳」の運用をガイドに明記）

**B-3.「年度末報告」シート（目玉機能）**

- 費目別の収入・支出集計、期首残高・期末残高を**全自動で整形済みレイアウトに表示**（A4縦1枚・印刷範囲設定済み）
- 学校でよくある「会計報告書」体裁（部活名・年度・作成日・顧問名の差し込みセル付き）
- GAS不要、全部数式。**「印刷するだけで職員会議・保護者会に出せる」**が売り文句

### C. 大会エントリー管理シート（完全版）

**C-1.「大会マスタ」シート**

| 列 | 項目 |
|---|---|
| A | 大会名 |
| B | 日程 |
| C | エントリー締切 |
| D | 会場 |
| E | 参加費（1人あたり） |
| F | 備考（集合時間など） |

- **締切アラート**：締切7日前で黄色、3日前で赤（条件付き書式。TODAY()比較）

**C-2.「エントリー」シート**

- 行=部員（名簿連動・在籍のみ）、列=大会（大会マスタ連動、最大10大会ぶんの列を用意）
- セルはチェックボックス。チェックした人数×参加費を列末で自動計算（会計簿への転記は手動運用。B-2と同じ二重計上防止の思想）
- 種目・部門の入力が要る競技向けに、チェックの隣に自由記入の細列を1本（テニスなら S/D の別など）

**C-3. GAS「📋 引率名簿を作る」**

- 大会を選ぶダイアログ→チェック済み部員の「学年・組・番号・氏名・担任」を新規シートに抜き出し、大会名・日程・会場・引率者欄付きのA4レイアウトで生成
- **管理職への引率届・生徒への配布そのままを想定**（ここが「60分→15分」の中身）

### D. 連絡テンプレ文例集

- スプシ内「連絡テンプレ」シートに1行1テンプレ（分類／件名／本文）。コピーして使う
- 差し込み箇所は【日付】【大会名】【金額】のように【 】で統一
- **基本版ライト10本**：欠席連絡のお願い／大会参加案内／集金案内／活動中止（雨天・警報）／活動時間変更／怪我発生の一報／面談・大会引率のお願い／夏季休業中の活動予定／体験入部案内／引退セレモニー案内
- **完全版で+15本**：管理職宛（活動計画書・大会引率届・会計報告添え文）／年間保護者会資料の骨子／新入部員保護者向け説明文／遠征時の持ち物・集合案内／未納の個別リマインド（角が立たない文面）など
- 文面ドラフトはコトハに依頼予定（現役顧問の実文面をベースに。ここも当事者性が商品力になる）

## 4. 「教員が使える」ためのUX要件

1. **初期設定は名簿貼り付けだけ**
   - 「はじめに」シートの手順は3ステップ以内：①このスプシをコピー ②「名簿」に部員を貼る ③メニュー「🎾部活キット」を1回実行（GAS承認）
   - GAS承認画面（「このアプリは確認されていません」）は非IT教員の最大の脱落ポイント。**note記事にスクショ付きで「この画面が出たら正常です」と先回り**で書く
2. **入力箇所を色で固定**：入力してよいセル=薄クリーム色、自動セル=白、で全シート統一。「色のついたところだけ触ってください」の1ルール
3. **個人情報の扱い（商品の信頼性に直結）**
   - 「はじめに」シートに明記：氏名・出欠は個人情報。**各校の情報管理規程に従い、学校アカウントの共有ドライブ内で運用すること／リンク共有は「制限付き」のまま使うこと／校外・私用アカウントへの持ち出し不可**
   - 保護者連絡先の列は**あえて設けない**（v1は顧問1人運用で不要。漏えいリスクを構造的に下げる）。備考列にも「最小限に」の注記
   - 販売ページにも「本テンプレは架空データ入り。生徒情報は購入者の管理下でのみ入力」と記載（教員相手の商売として、ここが甘い商品は選ばれない）
4. **学校の共有ドライブ想定**：複数顧問（正・副）での同時編集OKな設計（編集範囲保護は名簿・設定のみ）。「引き継ぎ」を見据え、年度更新手順（名簿貼り替え→出欠簿リセット）をガイドに1節設ける
5. **スマホ耐性**：毎日触るのは「今日の出欠」だけ、その1枚は列6本・プルダウン中心でスプシアプリから操作可能に。コート脇でスマホで付けられる＝現役顧問ならではの要件
6. **壊れた時の保険**：「設定」シートに「初期状態に戻すには→配布リンクからコピーし直し、名簿だけ貼り直す」と明記（サポート問い合わせを構造的に減らす）

## 7. 社長に決めてもらう論点（3つ）

**論点1：会計簿は完全版だけでよいか？**
- 本設計は「基本版=出欠のみ／会計・大会は完全版」の切り分け。¥1,980と¥3,980の差をはっきりさせ、完全版に誘導する狙い。
- 対案：会計簿を基本版に入れて基本版を¥2,980に上げる手もある（コギト氏の主力価格帯）。ただし完全版との差が「大会管理だけ」になり、アップグレードの魅力が薄れる。
- **サトルの推し：本設計どおり（会計は完全版）。** → 1. 賛成 ／ 2. 会計も基本版に入れる

**論点2：出欠区分プルダウンの標準セットは「出席／欠席／遅刻／早退／公欠／見学」でよいか？**

**論点3：販売名義をどうするか（実名 or 匿名の発信者名）**
