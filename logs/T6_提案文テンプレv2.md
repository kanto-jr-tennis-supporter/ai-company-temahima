## v1からの改善点（なぜ作り直したか）

1. **実績が「納品済み」に昇格**：旧版は「現在開発しております」止まり。今は実案件として中身（プルダウン・自動色付け・自動転記・集計・手作業ほぼゼロ）まで書ける。
2. **2つ目の実績を追加**：入札参加資格申請書＋メール文面の自動作成GASウェブアプリ（開発・納品済み）。「1件だけの人」から「複数納品の人」へ。
3. **稼働条件を正直に**：旧版の「週20〜30時間」は実態より盛り気味。「本業あり・平日夜/週末中心・非同期歓迎」に統一。ここを盛ると受注後に必ず苦しくなる。
4. **「進め方の3ステップ」を標準装備**：発注者の一番の不安は「頼んで大丈夫か」。小さく作って広げる進め方を示すのが最も効く。
5. **冒頭は相手の募集文の引用から**：T10で手応えのあった型。「読んでいる感」が最初の1行で伝わる。

## 1. 基本テンプレ（フル版）

> 使いどころ：GAS・スプレッドシート自動化・業務効率化など、**本命の案件**。文字数制限がゆるく、じっくり読まれる案件向け。約700〜900字。

```
ご担当者様

はじめまして。Googleスプレッドシート×GAS（Google Apps Script）で、手作業のデータ集計・転記を自動化する仕組みづくりを行っております、【CWプロフィールの表示名】と申します。「【相手の募集文にある課題・背景の言葉をそのまま1つ引用】」という点を拝見し、まさに同じ状況の現場をご支援した経験があるため応募いたしました。

■ 同型の課題を解決した実績
直近では、営業会社様のテレアポ管理シートを業務委託で自動化し、納品しました。
・対応ステータスのプルダウン化・自動色付けで入力を統一
・条件に合う企業データを、同シート内の一覧と別ファイルへ自動転記
・コール数などの実績を自動集計
これにより、担当者の方が毎日行っていた転記・集計の手作業をほぼゼロにしています。ほかにも、入札参加資格の申請書類とメール文面を自動作成するGASウェブアプリを開発・納品した実績があります。いずれも課題の聞き取りから要件定義・実装・テスト・運用定着まで一人で担当しました。

■ 進め方のご提案
1. 【相手の業務】の現状フローと「時間を取られている作業」のヒアリング
2. 【提案する自動化の中身を1行で。例：貼り付けデータの自動分割ツール】を小さく作り、実際に使っていただく
3. 効果を確認しながら対象を広げ、手順書化・引き渡しまで整える
最初から大きな仕組みを作るのではなく、「確実に動いて手が減るもの」から積み上げる進め方をご提案します。

■ 稼働について（正直にお伝えします）
本業があるため、稼働は平日夜・週末が中心です。チャットでの非同期のやり取りを基本に、必要に応じて夜間・週末のオンラインMTGという形であれば、無理なく安定して対応できます。平日日中の即時のご返信は難しい場合がありますが、その分、報告・確認はこまめに行います。

■ 確認させていただきたい点
・【案件ごとの技術的な確認を1つ。例：元データの形式（Excel／スプレッドシート／CSV）はどれでしょうか】

【もし案件が自分の専門外の領域を含むなら：得意領域は「集計・転記業務の自動化」であり、◯◯は専門外である旨を1〜2文で正直に。含まないならこの行ごと削除】

ご検討のほど、よろしくお願いいたします。
```

### フル版の差し替え箇所チェックリスト

| 箇所 | 差し替える内容 | 注意 |
|---|---|---|
| 【CWプロフィールの表示名】 | プロフィールと一致させる | 不一致は不信のもと |
| 【相手の課題の引用】 | 募集文の言葉を**そのまま**1つ | 言い換えない。原文が最強 |
| 【相手の業務】【自動化の中身】 | 進め方ステップ1・2を案件仕様に | ここが汎用のままだとテンプレ感が出る |
| 【確認点】 | 技術的な質問を**1つだけ** | 2つ以上は相手の負担 |
| 【専門外の線引き】 | 必要な案件のみ | 統計解析・デザイン等を求められたら正直に |

## 2. ショート版

> 使いどころ：応募が殺到する人気案件・データ入力系・「シンプルな提案文希望」の案件。**実績1つ＋一言提案**で終わる。約200〜250字。長文は読まれずに埋もれるため、最初の2行で勝負する。

```
はじめまして。営業会社様のテレアポ管理シートをGASで自動化し、業務委託で納品した実績があります（自動転記・自動集計で、毎日の手作業をほぼゼロにしました）。

【相手の課題を1行で言い換え。例：貼り付け後のデータ分割】は、まさに直近で扱った処理と同型です。【一言提案。例：onEditトリガーを使えばシンプルでメンテしやすい形にできます】。

本業があるため稼働は平日夜・週末中心ですが、報告・確認はこまめに行います。まずは詳細をお聞かせください。
```

### ショート版の差し替え箇所

- 【相手の課題を1行で言い換え】：募集タイトルか本文冒頭の言葉を使う
- 【一言提案】：技術ワードを**1箇所だけ**入れると「分かっている人」に見える（onEditトリガー、IMPORTRANGE、フォーム連携など）。自信がない技術名は書かない
- 応募必須項目（稼働時間・PC環境など）が指定されていたら、最終段落に**実際の値**で追記

## 3. 使い方メモ（5項目）

1. **使い分けの基準**：GAS・自動化・効率化など「仕組みを作る」案件＝フル版。データ入力・転記など「作業を請ける」案件や応募20件超の人気案件＝ショート版。迷ったらショート版（読まれることが先決）。
2. **相手の募集文の言葉を拾う**：冒頭の引用は要約せず原文のまま。応募必須項目（稼働時間・環境・経験の明記など）は見落とすと即落選なので、応募前に募集文を2回読む。
3. **数字は「事実の数字」だけ入れる**：「手作業をほぼゼロに」は事実なのでOK。削減時間・件数を出せる案件なら1つ添えると強い。**出せない数字・盛った稼働時間は書かない**（受注後の自分を苦しめる）。
4. **職業名は書かない**：「本業があるため」で止める。「元教員」等のフレーズも文面には入れない（直近ドラフトで一貫済み）。面談で聞かれたら正直に答える。
5. **送信は必ず社長の手で**：このテンプレで作った文面は毎回ドラフト扱い。名乗り・稼働時間・確認点を案件に合わせて確定させ、社長の目視確認後に応募する。
