## パート1：構築（約10分・PCで）

> ⚠️ 匿名販売の方針のため、**将来の配布用マスターは実名が出ないGoogleアカウントで作る**必要があります
> （コピー配布時に「オーナー名」が購入者に見えるため）。
> 今回の検証は普段のアカウントでOKですが、発売前に匿名アカウントで作り直す前提でお願いします。

### STEP 1：新しいスプレッドシートを作る
1. ブラウザで https://sheets.new を開く（新規スプレッドシートが1枚できます）
2. そのまま次へ（名前は後で自動で付きます）

### STEP 2：コードを貼り付ける
1. メニューの「拡張機能」→「Apps Script」をクリック（新しいタブが開きます）
2. 左のエディタに最初から入っている数行（`function myFunction() {...}`）を**全部消す**
3. `logs/T12_部活キット_出欠管理.gs` の中身を**全部コピーして貼り付け**
4. 上の💾（保存）アイコンをクリック。プロジェクト名を聞かれたら「部活キット」など適当でOK

### STEP 3：setup を実行する
1. エディタ上部の関数選択プルダウン（「onOpen」等が出ている場所）で **`setup`** を選ぶ
2. 「▶ 実行」をクリック

### STEP 4：承認画面を進める（初回だけ。ここが最大のつまずきポイント）
実行すると承認を求められます。**この流れは購入者も全員通る道**なので、
スクショを撮りながら進めると、そのままnote記事の素材になります。

1. 「承認が必要です」→「権限を確認」をクリック
2. 自分のGoogleアカウントを選ぶ
3. **「このアプリは Google で確認されていません」と出ます → これが正常です**
   - 「詳細」をクリック →「部活キット（安全ではないページ）に移動」をクリック
4. 「許可」をクリック
5. 実行が続き、「セットアップが完了しました！🎾」のダイアログが出れば成功

> うまくいかないとき：
> - ダイアログが出ない → スプレッドシートのタブに戻ると出ていることがあります
> - エラーで止まった → もう一度 ▶実行（作成済みシートには触らない安全設計なので、再実行しても壊れません）

### STEP 5：できあがりを確認する（1分チェック）
スプレッドシートのタブに戻り、以下を確認：

- [ ] シートが6枚：**はじめに／名簿／今日の出欠／出欠簿／出欠サマリ／設定**
- [ ] 「名簿」に架空の見本20名（うち1名「引退」）
- [ ] 「今日の出欠」に**19名**が表示されている（引退の1名が消えていれば名簿連動OK）
- [ ] 「今日の出欠」B1に今日の日付、区分列にプルダウン（欠席/遅刻/早退/遅早）
- [ ] クリーム色のセル＝入力場所、になっている
- [ ] ページを再読み込みすると、メニューバーに「**🎾部活キット**」が出る（数秒待つ）

## パート2：動作検証（その場で5分）

1. **空欄のまま記録**：メニュー「🎾部活キット」→「✅ 今日の出欠を記録する」
   - [ ] 「出席(○) 19人」と報告が出る（**空欄＝出席**の確認）
   - [ ] 「出欠簿」に今日の日付列ができ、19人全員に○
   - [ ] クリア確認で「はい」→「今日の出欠」の入力欄が空に戻る
2. **欠席・遅刻を入れて上書き**：適当な2人に「欠席」「遅刻」＋理由を入力 → もう一度✅実行
   - [ ] 「すでにあります。上書きしますか？」の確認が出る →「はい」
   - [ ] 出欠簿で×は赤、遅は黄色になる。×のセルにマウスを乗せると理由がメモで見える
3. **過去の日付**：「今日の出欠」B1 に昨日の日付を手入力（例 `2026/7/6`）→ 誰か1人欠席にして✅実行
   - [ ] 出欠簿で昨日の列が**今日の列より左**（日付順の位置）に入る
   - [ ] クリア後、B1 が今日に戻っている
4. **サマリ確認**：「出欠サマリ」を開く
   - [ ] 記録日数・欠席・遅刻・出席率が入っている（出席率＝(記録日数−欠席)÷記録日数。遅刻・早退は出席扱い）
   - [ ] 上部の「記録状況」が「OK：最後の記録は …」になっている
5. **区分の追加**：「設定」シートの区分表の空き行に「公欠」を追加 →「今日の出欠」のプルダウンに増えるか
   - [ ] 増えたら、誰かを公欠にして記録 → 出欠簿に「公」（1文字目ルール）で入る

ここまで全部✅なら、実装どおり動いています。

## パート3：実運用テスト（3日間・設計書の工程3）

ご自身のテニス部の名簿で3日間、本番運用してください。**ここは社長にしか代われません**
（そして「現役顧問が自部で使っている」という販売文の根拠になります）。

- 準備：「名簿」の見本20名を消して、実名簿を貼り付け（学年・組・番号・氏名。在籍は「在籍」）
  - ⚠️ 実名簿を入れたファイルは**検証専用**。配布用マスターには絶対に流用しない
    （配布用は見本データのままの状態から別途コピーして作ります）
- 毎日：練習後に「今日の出欠」で休んだ人だけ区分を選ぶ →（PCで）✅記録。**所要時間をメモ**
- 見てほしい観点（赤入れメモをそのままください）：
  1. 現行の手作業（約10分）と比べて何分になったか
  2. 区分4つ（欠席/遅刻/早退/遅早）で足りたか。「これも要る」があったか
  3. スマホで入力→PCで記録、の流れは現実的か（コート脇での使用感）
  4. 出欠簿・サマリの見た目は「先生が職員室で開いて恥ずかしくない」か
  5. 文言でひっかかった所（「学校ではこう言わない」が商品価値になります）

## 知っておいてほしい仕様（設計判断のメモ）

| 項目 | 仕様 | 理由 |
|---|---|---|
| 空欄＝出席 | 転記時に空欄→○ | 社長決裁（§8）。テニス部の実運用 |
| 記号 | ×/遅/早/遅早。追加区分は記号欄が空なら1文字目 | 区分を増やしても壊れない汎用ルール |
| 出席率 | (記録日数−欠席)÷記録日数 | ×だけを欠席扱い。学校ごとの調整はカスタム受託の商材に |
| 連続3回欠席 | 出欠簿の右端3列がすべて×で⚠️ | 数式のみ（GAS不使用）で軽い |
| つけ忘れ | 最終記録から3日以上でサマリに⚠️ | 顧問の自己アラート |
| 記録の実行 | PCのブラウザのみ | GASのカスタムメニューはスマホアプリ非対応（正直な制約として「はじめに」に明記済み） |
| setup再実行 | 既存シートには一切触れない | 壊れたシートだけ削除→再実行で個別復旧できる |
| 年度更新 | 「出欠簿」+「出欠サマリ」を2枚セットで削除→setup | サマリの数式が出欠簿を参照しているため片方だけの削除はNG |
| 匿名性 | コード・見本データに実名・学校名・実在生徒名なし | 社長決裁（§8）。見本20名は完全な架空名 |

## よくあるトラブル

| 症状 | 対処 |
|---|---|
| メニュー「🎾部活キット」が出ない | ページ再読み込み→数秒待つ。それでも出なければApps Script画面で保存できているか確認 |
| 「今日の出欠」に部員が出ない | 名簿の「在籍」列が「在籍」になっているか。A3セルの数式を消していないか（消したら「今日の出欠」シートを削除→setup再実行） |
| 承認画面で「戻る」を押してしまった | もう一度 ▶実行すればやり直せます |
| シートを壊した | 壊れたシートを削除→「🛠 初期セットアップ」（出欠簿を消すときはサマリも一緒に） |
