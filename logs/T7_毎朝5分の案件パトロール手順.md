# T7：毎朝5分の案件パトロール手順書

- 作成：リサ（案件リサーチャー）／2026-07-07
- 使う人：山元社長（毎朝、通勤前や休み時間のスキマ5分）
- ねらい：クラウドワークスの新着から「①データ集計効率化」クラスの当たり案件を、掲載直後（応募5件以下のうち）に拾う。**新着への先行応募が採用率をいちばん上げます。**

> ⚠️ 正直な注記：私の作業環境からは crowdworks.jp に接続できない（403ブロック）ため、下のURLは**クラウドワークスの既知のURL形式に基づいて作成**しています。**初回だけ、社長が1本クリックして「検索結果が開くか・新着順になっているか」を確認してください。** もし開かない／並び順がおかしい場合は、サイトで一度手動検索して出たURLを教えてもらえれば、私が全リンクを作り直します。

**URLの仕組み（自分でキーワードを変えたいとき）**
- 基本形：`https://crowdworks.jp/public/jobs/search?search%5Bkeywords%5D=キーワード&order=new`
- `%5B` `%5D` は `[` `]`、`%20` はスペース。日本語はブラウザに直接打ってもOK（自動で変換されます）。
- `&order=new` を付けると新着順。付けないとおすすめ順（古い人気案件が混ざるので**必ず新着順で見る**）。
- ※2番だけ `order=new` を外してあります＝動作確認用の比較対象。開いてみて並び順が変なら、サイト上の「新着順」タブを押した後のURLをコピーして差し替えてください。

## 2. 30秒足切り基準（タイトルと報酬だけで判断）

検索結果の一覧画面で、**本文は開かずに**タイトル・報酬・応募数だけで振り分けます。1件10秒、1ページ30秒。
（T9の5分チェックリストの圧縮版。迷ったら本体 `logs/T9_クラウドワークス6件精査.md` の付録へ）

### 即×ワード（タイトルにあったら開かない）

- **資格系**：税理士／社労士／行政書士／「有資格者限定」（T9③型。無資格では違法になる領域）
- **常駐・拘束系**：毎日対応／日次報告／即レス／常駐／シフト／「平日日中」／カスタマーサポート・CS／コールセンター（T9②⑤型。教員の生活と構造的に両立不可）
- **単純作業系**：データ入力／文字起こし／目視確認／チェック作業／モニター／アンケート（T9④型。スキルが資産にならない時間切り売り）
- **危険系**：フォーム送信／営業代行 送信／「ボタンを押すだけ」／「1日5分で○円」／不自然に楽で安い（T9⑥型。スパム加担リスク。**CWアカウントと評価を守るため関わらない**）
- **守備範囲外**：Web制作／LP／デザイン／動画編集／Python環境構築／AWS／DB構築

### 報酬の足切り

- **時給制**：1,500円未満は× ／ 1,500円以上でも「月○時間拘束」型は原則見送り（納品型を優先）
- **固定報酬**：GAS・スプシ単発は**1〜5万円が相場帯**。5,000円未満は初実績づくり目的のときだけ○
- **「要相談」「見積もり式」**：タイトルが◎ワードなら開いてよい（自分で値付けできるチャンス）

### 開く価値あり（◎ワード：1つでもあれば本文へ）

- GAS／Apps Script／スプレッドシート×（自動化・転記・集計・連携・関数）／Googleフォーム／請求書 自動／管理シート 作成・改善
- **さらに「応募5件以下＋掲載直後」なら最優先**（20件超は埋もれやすい。T9チェックリストQ5）

## 3. 有望案件の送り方テンプレ（見つけたら私にコピペで投げる）

◎案件を見つけたら、その場で応募せず**まず私（チーム）に精査依頼**してください。T9で回った方式です。
作業環境からCWのページが開けないため、**URLだけでなく本文コピペが必須**です。

Claude Code のチャットにそのまま貼り付け：

```
リサ、案件の精査をお願い。T9のチェックリストで判定して。

【1件目】
URL：https://crowdworks.jp/public/jobs/XXXXXXXX
応募数：○人（見えた数字）／掲載日：○月○日（見えれば）
--- 以下、案件ページの本文を全文コピペ ---
（タイトル〜報酬〜仕事の詳細〜応募条件まで、ページを全選択コピーでOK。余計な部分は私が読み飛ばします）
---

【2件目】（あれば同じ形式で続ける）
```

- **コピーのコツ**：案件ページで Cmd+A → Cmd+C の全選択コピーで十分。整形不要。
- 私からの返しは T9 と同じ形式：**○条件付きGO／×見送り＋理由＋（GOなら）応募文で押すポイント**。
- GO判定が出たら、そのままコトハに応募文作成を回せます：「コトハ、この案件の応募文を書いて。テレアポシート実績を冒頭に」

## 4. 週次の振り返り指標（金曜か日曜に2分）

- **応募数**：週3〜5本が目安。0〜1本の週は「案件がない」のではなく足切りが厳しすぎるサイン。◎ワードの範囲か☆URLの巡回頻度を広げる。
- **返信率（返信÷応募）**：**3割前後あれば応募文は機能しています**。5本応募して返信0なら案件選びではなく文面の問題——コトハに応募文の型を見直させる。
- 数字は応募のたびにチャットで一言（「T○、応募した」）くれれば、私たちが売上ボードのファネルに記録します。

## 毎朝のルーティン（まとめ：計5分）

1. ★1〜★3のブックマークを順に開く（1分）
2. 一覧を「即×ワード」でスキャン、◎ワード×応募5件以下を探す（3分）
3. 見つけたら §3 テンプレでコピペ送信して閉じる（1分）。無ければ即終了——**毎日ゼロでも正常です。週に1〜2本の当たりを逃さないための網張り**なので。
