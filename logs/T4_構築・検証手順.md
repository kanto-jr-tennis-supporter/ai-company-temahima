# T4 入札申請書アプリ プロトタイプ 構築・検証手順

対象：サンプル⑤（文部科学省・様式1-1／1-2／2 の3シートのみ＝最小構成）
コード：`logs/T4_入札申請書アプリ_プロトタイプ.gs`
設計書：`logs/T4_入札申請書アプリ_設計書v1.md`（サトル作成）

社長が手を動かすのは①②③④の4ステップです。1つずつ進めてください。

## ①サンプル⑤.xlsx をスプレッドシートにインポートする

1. Google スプレッドシートを新規作成する（名前は何でもよい。例：「入札申請書アプリ（文科省版）」）。
2. メニュー「ファイル」→「インポート」を開く。
3. 「アップロード」タブから `logs/T4_入札サンプル/サンプル⑤.xlsx` を選択する。
4. インポートの選択肢で **「シートを挿入する」** を選ぶ（既存シートを置き換えない）。
5. 「インポート」を実行する。
6. インポート後、シート名が **「様式1-1」「様式1-2」「様式2」** の3つになっていることを確認する
   （インポート時に自動で別名（例：「様式1-1 (1)」）になっていたら、その名前を正確に
   「様式1-1」「様式1-2」「様式2」に手動でリネームしてください。GASはこの名前を前提に動きます）。
7. デフォルトで作られる空の「シート1」は削除してよい。

---

## ②GASを貼り付けて setup() を実行する

1. メニュー「拡張機能」→「Apps Script」を開く。
2. 最初から入っている `Code.gs` の中身を全て削除し、`logs/T4_入札申請書アプリ_プロトタイプ.gs`
   の中身をすべてコピー＆貼り付けする。
3. 画面上部の保存アイコン（💾）で保存する。
4. 関数選択のプルダウンで **`setup`** を選び、▶実行ボタンを押す。
5. 初回は「承認が必要です」という画面が出る。Googleアカウントを選び、
   「詳細」→「（安全ではないページに移動）」→「許可」と進めて承認する
   （これは自分で作ったスクリプトなので安全。Googleの定型警告です）。
6. 「マスターシートが見つかりません」というエラーが出た場合 → ①のシート名リネームが
   できていません。シート名を直してから、もう一度 `setup` を実行してください。
7. 「セットアップが完了しました」のアラートが出れば成功。スプレッドシートに戻ると
   「入力ガイド」「入力フォーム」「設定」の3シートが追加されているはずです。

## ④転記・○配置の動作確認

1. メニューバーに「入札申請書ツール」という独自メニューが増えているはずです
   （出ていなければスプレッドシートを再読み込みしてください）。
2. 「入札申請書ツール」→「② マスターへ転記する」を実行する。
3. 完了アラートで「テキスト転記: n件 / 楕円配置: 1件 / 文字分解転記: n件」と出ることを確認する。
4. 「様式1-1」シートを開き、目視で以下を確認する：
   - 「新規」の文字のまわりに**赤い楕円の画像**が重なって配置されているか（○の代わり）。
   - 商号・住所・役職・氏名・許可番号（ハイフンの前後）が、正しいマス目に入っているか。
   - フリガナ（商号・住所・代表者氏名の3箇所）が、ラベル「フリガナ」の右側に
     1文字ずつ並んで入っているか。**ここは特に重要な確認ポイントです**（下記「既知の注意点」参照）。
5. もう一度「② マスターへ転記する」を実行しても、楕円が二重に増えないこと（前の楕円が
   消えて置き直されること）を確認する。

## 既知の注意点（正直に書きます）

### 1. フリガナの「1文字1セル」は座標を推測で当てています

サンプル⑤を Python（openpyxl）で実測したところ、フリガナの記入欄そのものは
**罫線のない1個の結合セル**（例：商号ふりがなは `X23:GA23`）であることが分かりました。
「1文字＝1セルの明確な箱」が罫線で裏付けられているわけではありません。

ラベル文字「フ・リ・ガ・ナ」が2列おき（列F・H・J・L）に置かれていて、その結合セルの
開始位置（X列）がちょうどこのピッチの続き（列N・P・R・T・V＝5マス）の直後に来ることから、
「この5マスが本来の1文字入力欄だろう」と推測して実装しています。**確実な保証ではありません。**

→ ④の手順4で、フリガナが変な位置にズレていないか必ず目視確認してください。
もしズレていたら、`logs/T4_入札申請書アプリ_プロトタイプ.gs` の `FIELD_CONFIG` にある
`shago_furigana` / `jusho_furigana` / `daihyo_furigana` の3行を `kind: 'text'` に変更し、
転記先セルを結合セル側（`X23` / `X18` / `X31`）に直せば、1文字ずつではなく
フリガナ全体を1つの文字列としてそのまま書き込む方式に切り替えられます（そちらの方が確実です）。

### 2. ○の図形は「本物の図形」ではなく画像で代用しています

Google Apps Script の SpreadsheetApp（スプレッドシート用のAPI）には、Google スライドの
`insertShape(OVAL)` に相当する「図形を直接プログラムで挿入する機能」がありません
（2026年7月時点で確認）。そのため、あらかじめ用意した**赤い楕円のPNG画像**を
`Sheet.insertImage()` で該当セルの上に自動配置する方式にしています。

- 見た目は「○で囲んである」ように見えますが、実体は画像です。印刷・PDF化すれば
  問題なく○として見えます。
- セルの行列幅から自動でサイズを計算して重ねていますが、微妙にズレる場合は
  コード内の `insertOvalMark_` 関数の `setWidth` / `setHeight` の値を手で微調整してください。

### 3. マスターシートの保護について

`setup()` 実行後、様式1-1／1-2／2 は「保護」がかかり、人間が手で書き込めなくなります
（GASの `転記する` だけが書き込めます）。もし内容を作り直したい場合は、対象シートを開き、
右上の鍵アイコンから保護を一旦解除してください。
