## パート1：構築（約10分・PCで）

> ⚠️ 匿名販売の方針のため、**将来の配布用マスターは実名が出ないGoogleアカウントで作る**必要があります
> （コピー配布時に「オーナー名」が購入者に見えるため）。今回の検証は普段のアカウントでOKです。

### STEP 1：新しいスプレッドシートを作る
1. ブラウザで https://sheets.new を開く
2. そのまま次へ（名前は後で自動で付きます）

### STEP 2：コードを貼り付ける
1. メニューの「拡張機能」→「Apps Script」をクリック
2. 左のエディタに最初から入っている数行を**全部消す**
3. `logs/T13_部活キット_会計簿.gs` の中身を**全部コピーして貼り付け**
4. 上の💾（保存）アイコンをクリック。プロジェクト名は「部活キット」など適当でOK

### STEP 3：setup を実行する
1. エディタ上部の関数選択プルダウンで **`setup`** を選ぶ
2. 「▶ 実行」をクリック

### STEP 4：承認画面を進める（初回だけ）
T12と同じ流れです。
1. 「承認が必要です」→「権限を確認」→ 自分のGoogleアカウントを選ぶ
2. 「このアプリは Google で確認されていません」→「詳細」→「（安全ではないページ）に移動」
3. 「許可」をクリック
4. 「セットアップが完了しました！🎾」のダイアログが出れば成功

### STEP 5：できあがりを確認する（1分チェック）
スプレッドシートのタブに戻り、以下を確認：

- [ ] シートが**9枚**：はじめに／名簿／今日の出欠／出欠簿／出欠サマリ／**会計簿／部費徴収／年度末報告**／設定
- [ ] 「設定」シートに H列「費目一覧（収入向け）」・I列「費目一覧（支出向け）」が入っている
- [ ] 「設定」シートのA18に「顧問名」の入力欄がある
- [ ] ページを再読み込みするとメニュー「🎾部活キット」が出る（会計まわりはボタン不要。数式で自動計算されます）

## パート2：動作検証（会計まわり・15分）

出欠管理の検証（記録・上書き・過去日付・サマリ・区分追加）は `logs/T12_構築・検証手順.md` の
パート2をそのまま実施してください。ここでは追加した会計まわりだけを確認します。

### 2-A．会計簿（記帳と残高の自動計算）

1. 「会計簿」シートを開く。B1「期首残高」が 0 になっていることを確認
2. 3行目に記帳してみる：日付＝今日、種別＝**収入**（プルダウン）、費目＝**部費**（プルダウン）、
   内容＝「テスト記帳」、金額＝30000、領収書＝あり
   - [ ] 行全体がうす緑になる（収入＝緑の確認）
   - [ ] F3「残高」に **30000** が自動で入る
3. 4行目：種別＝**支出**、費目＝**消耗品**、金額＝5000、領収書＝**なし**
   - [ ] 行全体がうす赤になる（支出＝赤の確認）
   - [ ] F4「残高」に **25000** が自動で入る（30000－5000）
   - [ ] G4「領収書：なし」のセルが黄色になる（支出なのに領収書なし＝注意表示）
4. 5行目：種別＝支出、費目＝大会参加費、金額＝40000（B1の期首残高より大きい額であえて赤字にする）
   - [ ] F5「残高」がマイナスになり、赤字・太字で表示される
5. C列（費目）で「設定」シートに無い文字を直接入力しようとする
   - [ ] プルダウン以外の値は入力できない（厳格設定の確認）

### 2-B．部費徴収（未納者一覧・入金済み合計）

1. 「部費徴収」シートを開く。D13「第1回」・D14「3000」が見本で入っていることを確認
2. 上部の「未納者一覧」1行目に「第1回（〆切未設定）」＋部員全員の名前が並んでいることを確認
   （まだ誰もチェックしていないので全員未納の状態）
3. 19行目（1人目）のD列チェックボックスにチェックを入れる
   - [ ] そのセルの赤色が消える
   - [ ] 「未納者一覧」1行目からその部員の名前が消える
   - [ ] D16「入金済み合計」が 3000 になる（1人×3000円）
   - [ ] D17「未納人数」が1人減る
4. D15「締切」に日付を入れる（例：来月1日）
   - [ ] 「未納者一覧」1行目の表示が「第1回（〆切◯/◯）」に変わる
5. E13〜E15（第2回）を空欄のままにしておく
   - [ ] 「未納者一覧」2行目が「（この回は未使用です）」のままであることを確認（未設定の回は対象外）

### 2-C．年度末報告（印刷レイアウトと自動集計）

1. 「年度末報告」シートを開く
   - [ ] F2「対象年度」に今年度の開始年（例：2026）が自動で入っている
   - [ ] A2「部活名」に「設定」シートB16の内容（初期値「○○部」）が表示される
2. 2-Aで記帳した内容が反映されているか確認
   - [ ] 「部費」の行に 30000 が入る
   - [ ] 「消耗品」の行に 5000、「大会参加費」の行に 40000 が入る
   - [ ] 「収入合計」30000／「支出合計」45000／「差引収支」－15000 になっている
   - [ ] 「期首残高」0／「期末残高」－15000（会計簿F5の残高と一致）になっている
3. 印刷プレビューを開く（ファイル→印刷）
   - [ ] 「現在のシートのみ」を選ぶと、A4縦1枚に収まっている
   - [ ] 部活名・年度・作成日・収支・残高が読める体裁になっている
4. 「設定」シートA18「顧問名」に名前を入れてみる
   - [ ] 「年度末報告」F3「顧問名」に反映される

ここまで全部✅なら、会計まわりも実装どおり動いています。

## 知っておいてほしい仕様（設計判断のメモ）

| 項目 | 仕様 | 理由 |
|---|---|---|
| 会計簿の残高 | B1（期首残高）を起点に、1つ上の行の残高＋当該行の収支を積み上げる累計式 | GAS不要・壊れにくい。行を上から埋めていく前提（途中の行だけ埋める使い方は非対応） |
| 費目プルダウン | 「設定」シートH列（収入向け）・I列（支出向け）を1つの範囲として参照 | 種別ごとに厳密に出し分けず、費目名だけで判断。壊れにくさ優先（依存関係のあるプルダウンはGAS onEdit実装が必要になり複雑・壊れやすいため見送り） |
| 種別・費目・領収書 | いずれも「入力ミス防止」で厳格プルダウン（setAllowInvalid(false)） | 費目集計・年度末報告のSUMIFS条件に使う文字のため、表記ゆれがあると集計が狂う |
| 部費徴収の未納者一覧 | FILTER関数を呼ぶ前にCOUNTIFSで0件かどうかを判定 | FILTERは対象0件だと#N/Aエラーになる仕様のため、出欠サマリ（T12）と同じ安全策を踏襲 |
| 部費徴収の合計転記 | 会計簿へは自動転記しない。金額を確認して手動で1行記帳する運用 | 二重計上事故の防止（社長決裁書・設計書§3-Bのとおり） |
| 年度末報告の年度絞り込み | F2（対象年度・開始年）とSUMIFSの日付条件で、会計簿の中からその年度分だけを集計 | 会計簿は年度をまたいで記帳を続ける設計のため、年度末報告側で期間を絞る必要がある |
| 年度末報告の期首残高 | 会計簿!B1 ＋（対象年度の開始日より前の収支の累計） | 会計簿をリセットせずに何年も使い続けても、その年度単体の期首残高が正しく出るようにするため |
| 印刷レイアウト | GAS不使用。全部数式で作成し、シートのサイズを内容ぴったりに絞ってある | Apps Scriptには印刷範囲を直接指定するAPIが無いため、「ファイル→印刷→現在のシートのみ」で自然にA4に収まるようシートの行列数を絞ることで対応 |
| 匿名性 | コード・見本データに実名・学校名・実在生徒名なし。部費徴収の見本金額（3,000円）も一般的な相場感の架空値 | 社長決裁（T11§8）を踏襲 |

## 既知の制約・今後の課題

- **T12だけで運用していた既存スプレッドシートへの後乗せ**：このファイルを既存のT12ベースのシートに貼り直して
  `setup` を再実行しても、「設定」シートはすでに存在するため中身は更新されません（安全設計として、既存シートに
  一切触らない仕様のため）。その場合は「設定」シートのH列・I列（費目一覧）とA18（顧問名）を手動で追加するか、
  「設定」シートを削除してから`setup`を再実行してください（区分のカスタマイズをしていた場合は初期値に戻ります）。
- 大会エントリー管理（C）・連絡テンプレ文例集（D）は本工程には含まれていません（設計書§6の工程どおり、別タスクで実装予定）。
- 部費徴収の徴収回は最大8回（第1回〜第8回）までの対応です。9回以上必要な部活は「設定」シートの費目一覧と
  同様、列を追加する形でのカスタム対応（ココナラ）を案内してください。

## よくあるトラブル

| 症状 | 対処 |
|---|---|
| 費目プルダウンに何も出ない | 「設定」シートH4:I8に費目が入っているか確認。空にすると選べなくなります |
| 会計簿の残高が全部空欄 | A列（日付）が未入力だと残高も空欄になる仕様です。日付を入れてください |
| 部費徴収のチェックが反映されない | チェックボックスは名簿の「在籍」の人だけ表示されます。「名簿」の在籍列を確認してください |
| 年度末報告の金額が0のまま | F2「対象年度」と、会計簿の日付が同じ年度になっているか確認（年度開始月は「設定」シートB17） |
| 年度末報告の部活名が「○○部」のまま | 「設定」シートB16を書き換えてください |
