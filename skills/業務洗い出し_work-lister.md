# 🧭 スキル：業務洗い出し（work-lister）— 担当：サトル

クライアント（または社長自身）と対話しながら業務を洗い出し、スプレッドシートに自動記録するツール。
「何を効率化・自動化すべきか」の出発点を作る、要件定義の前工程。

- **場所**: `tools/work-lister`（キット内。実体プロジェクトへのリンク。無い場合は導入オプション）
- **詳細な使い方**: `tools/work-lister/CLAUDE.md`（スキル定義込み）を読むこと

## 主要コマンド（`tools/work-lister` で実行）

```bash
cd tools/work-lister
node tools/init-spreadsheet.mjs                       # スプシ接続テスト（--id でID保存）
node tools/add-work.mjs '{"name":"...","flow":"..."}'  # 洗い出した業務を1件追加
```

## 進め方（サトルの仕事の型）

1. ヒアリングは**一度に2〜3問まで**。「毎日/毎週やっている作業」「時間がかかる作業」「ミスが起きやすい作業」から聞く。
2. 聞き出した業務を1件ずつ `add-work.mjs` でスプシに記録（業務名・流れ・頻度・所要時間）。
3. 全部出たら「自動化インパクト順」に並べた要約を作り、`logs/業務洗い出し_◯◯.md` に保存。
4. 上位の業務は「ハックに自動化の試作を頼む」流れを次のおすすめとして添える。

## 前提（動かない時はここを確認）

- `.env` と Google OAuth 認証済みが必要（`node auth-google.mjs`）。
- スプシ連携が落ちていても、**洗い出し〜整理は Markdown だけで完遂できる**。仕事を止めない。

## 納品ルール

- 業務一覧の要約を `logs/` に保存。スプシURLがあれば報告に含める。
