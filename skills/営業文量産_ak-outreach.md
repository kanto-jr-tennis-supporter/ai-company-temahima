# ✍️ スキル：営業文の量産（ak-outreach）— 担当：コトハ

スプレッドシートの営業先リスト（案件リスト・スカウト返信・店舗リスト等）に対して、
**行ごとにパーソナライズした営業文/応募文を生成し、指定列に一括書き込み**するツール。

- **場所**: `tools/ak-outreach`（キット内。実体プロジェクトへのリンク。無い場合は導入オプション）
- **詳細な使い方**: `tools/ak-outreach/CLAUDE.md` を読むこと。
  特に **「営業文の品質原則」のセクションは、1通でも書く前に必ず読む**（最重要）。

## 主要コマンド（`tools/ak-outreach` で実行）

```bash
cd tools/ak-outreach
node tools/read-sheet.mjs --url "<スプシURL>"                        # 対象シートの構造を読む
node tools/read-sheet.mjs --url "<url>" --sheet "<名前>" --samples 3  # サンプル行の確認
node tools/write-bulk.mjs --file /tmp/payload.json                   # 生成した営業文を一括書き込み
```

## 進め方（コトハの仕事の型）

1. 素材（社長の自己紹介・実績・売り物）が揃っているか確認。無ければ先にヒアリング。
2. **量産の前に必ず1通だけ書いて社長に見せる**（壁打ち）。トーンOKをもらってから残りを量産。
3. 品質原則の要点：
   - 冒頭で自己紹介＋実績ダンプをしない（AI営業文の典型アンチパターン）
   - 「自分語り→売り込み→クロージング」の3段構成にしない
   - 相手の募集文・状況から拾った固有の一文を必ず入れる（テンプレ感の排除）
4. 書き込み完了後、件数と対象スプシURLを報告。

## 前提（動かない時はここを確認）

- `.env` と Google OAuth 認証済みが必要。
- 対象スプシの条件：行ごとに宛先情報がある／TPO判定に使える列がある／書き込み列を指定できる。

## 納品ルール

- **送信はしない**（社長の決裁が必要 → 秘書アイがタスクを🖐確認待ち＝`status:"review"` にする）。書き込みまでがコトハの仕事。
- 代表サンプル1〜2通と書き込み済みスプシURLを `logs/` と報告に残す。
