# 🔍 スキル：案件探索＋応募文生成（job-scout）— 担当：リサ

複数の案件サイト（Wantedly / YOUTRUST / CrowdWorks / ココナラ / ランサーズ / Workship / SOKUDAN / 複業クラウド）を
横断検索して案件を一覧化し、**プロフィール×応募テンプレートから応募文も生成する**ツール。
スプレッドシートを設定画面・結果置き場として使う。

- **場所**: `tools/job-scout`（キット内。実体プロジェクトへのリンク。無い場合は導入オプション）
- **詳細な使い方**: `tools/job-scout/CLAUDE.md` を読むこと（各スキルの詳しい手順はすべてそちらが正）

## できること（スキル一覧）

| スキル | 内容 |
|---|---|
| Sheet Scout（案件探索） | スプシの設定条件で8サイト横断検索 →「案件一覧」シートに補充 |
| 応募文生成 | 指定IDの「募集詳細」×「プロフィール」A1×「応募テンプレート」A1 で応募文を生成 → N列「応募文メモ」に書き込み（**応募の送信はしない**） |
| プロフィール作成 | ヒアリング →「プロフィール」シート A1 に書き込み（応募文生成の材料） |
| 応募テンプレート作成 | 過去に通った応募文等からテンプレ化 →「応募テンプレート」シート A1 に書き込み |
| URL一括オープン | 応募文メモあり&未確認の案件URLを順次新規タブでオープン |
| ステータス更新 | 「1と3は応募した」をパースして 未確認/応募した/見送り を一括更新 |

## 主要コマンド（`tools/job-scout` で実行）

```bash
cd tools/job-scout
node run-from-sheet.mjs            # 案件探索（メイン）。スプシの設定条件で横断検索→スプシに書き込み
node all.mjs --format=markdown     # スプシなしで生の探索結果をMarkdownで取得（クイック調査向け）
node open-urls.mjs --ids=1,3,5     # 指定IDの案件URLをブラウザで順次オープン
node update-status.mjs --applied=1,3 --skipped=5   # 応募した/見送りのステータス一括更新
```

※ プロフィール作成・応募テンプレート作成・応募文生成は対話型スキル（エージェントがヒアリング→Sheets APIでA1や応募文メモ列に書き込む）。手順は `tools/job-scout/CLAUDE.md` に従う。

## 前提（動かない時はここを確認）

- `.env` に `SPREADSHEET_ID`、Google OAuth 認証済み（`node auth-google.mjs`）が必要。
- **認証が未設定でも `all.mjs --format=markdown` は動く**。スプシ連携が落ちていたら、
  まず `all.mjs` で結果を取り、`logs/案件リサーチ_◯◯.md` に保存して納品する（仕事を止めない）。

## 納品ルール

- 探索結果の**要約**（おすすめ順・なぜ社長に向いているか一言付き）を `logs/` に Markdown 保存。
- スプシに書き込んだ場合は、そのURLを報告に含める（秘書アイが `deliverables.url` に入れてオフィスに出す）。
- 有望案件が見つかったら「コトハに営業文を書かせる」流れを次のおすすめとして添える。
